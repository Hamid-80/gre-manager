# مدیریت پیشرفته تونل GRE (Advanced GRE Tunnel Manager)

یک اسکریپت بش (Bash) سبک، قدرتمند و تعاملی برای ساخت، حذف و مدیریت آسان تونل‌های GRE در سرورهای لینوکس (اوبونتو و دبیان).

این اسکریپت به صورت خودکار تنظیمات تونل را از طریق سرویس سیستمی `systemd` ماندگار می‌کند (تا با ریبوت سرور پاک نشود) و همچنین دارای یک ابزار داخلی قدرتمند برای پاک‌سازی و رفع قفل‌های شبکه (کارت‌های شبکه شبح یا Ghost) بدون نیاز به ریبوت کردن سرور است.

## 🚀 ویژگی‌های کلیدی

- **رابط کاربری تعاملی:** منوهای رنگی و منظم برای مدیریت سریع و بدون خطا.
- **تشخیص خودکار آی‌پی:** شناسایی هوشمند و خودکار آی‌پی عمومی سرور شما.
- **اتصال پایدار و ماندگار:** لود شدن خودکار تونل‌ها بلافاصله پس از روشن شدن یا ریبوت سرور.
- **سیستم رفع مشکل بدون ریبوت:** مجهز به ابزار فلاش (Flush) عمقی برای باز کردن کارت‌های شبکه قفل‌شده در کرنل لینوکس.
- **پاک‌سازی ۱۰۰ درصدی:** حذف کامل سرویس‌های سیستمی، اینترفیس‌ها و کش روتینگ در زمان حذف تونل.

## 📖 راهنمای تنظیمات تونل بین دو سرور

برای اتصال دو سرور (مثلاً سرور ایران و سرور خارج)، منطق وارد کردن آی‌پی‌ها باید به صورت آینه‌ای (قرینه) باشد:

### تنظیمات سرور A (سرور اول)

* **نام کارت شبکه (Interface Name):** `gre1` (حتماً از نام‌های استاندارد استفاده کنید و حروف تک‌حرفی مثل `h` نزنید).
* **آی‌پی عمومی سرور مقابل (Remote Public IP):** `[آی‌پی عمومی سرور B]`
* **آی‌پی عمومی این سرور (Local Public IP):** دکمه `Enter` را بزنید تا خودکار شناسایی شود.
* **آی‌پی داخلی تونل (Local Tunnel IP):** `10.10.1.1`
* **آی‌پی داخلی سرور مقابل (Remote Tunnel IP):** `10.10.1.2`
* **ساب‌نت ماسک (Subnet Mask):** دکمه `Enter` را بزنید (پیش‌فرض: 30).

### تنظیمات سرور B (سرور دوم)

* **نام کارت شبکه (Interface Name):** `gre1`
* **آی‌پی عمومی سرور مقابل (Remote Public IP):** `[آی‌پی عمومی سرور A]`
* **آی‌پی عمومی این سرور (Local Public IP):** دکمه `Enter` را بزنید تا خودکار شناسایی شود.
* **آی‌پی داخلی تونل (Local Tunnel IP):** `10.10.1.2` *(برعکس سرور اول)*
